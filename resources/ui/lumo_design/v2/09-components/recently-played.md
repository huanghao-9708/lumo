# Recently Played

> 最近播放记录页面，比全部歌曲多一个"播放时间"列。

## Anatomy

```
┌──────────────────────────────────────────────────────────┐
│  # │ ♥ │ 标题            │ 艺术家      │ 专辑    │ 播放时间 │
├──────────────────────────────────────────────────────────┤
│  01 │ ♥ │ Track Title    │ Artist      │ Album   │ 今日 10:30 │
│  02 │   │ Another Track  │ Artist      │ Album   │ 昨日 22:15 │
│  03 │   │ Old Track      │ Artist      │ Album   │ 2026-06-15 │
├──────────────────────────────────────────────────────────┤
│  X 首歌曲 · 总计 X 天 X 小时 X 分钟          双击播放    │
└──────────────────────────────────────────────────────────┘
```

## 播放时间格式化规则

| 时间范围 | 格式 | 示例 |
|---|---|---|
| 今天 | 今日 HH:MM | 今日 10:30 |
| 昨天 | 昨日 HH:MM | 昨日 22:15 |
| 更早 | YYYY-MM-DD | 2026-06-15 |

## Footer Status

底部统计行显示总播放时长（不同于全部歌曲的简单计数）：
```
总计 X 天 X 小时 X 分钟
```

## 后端依赖

- `TrackDTO.last_played_at` 字段（Rust + TS）
- `get_recently_played` SQL 查询返回 `t.last_played_at`
- `mapTrackDTO` 映射为 `Track.playedAt`
