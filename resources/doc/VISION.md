# Lumo（轻音）— 产品愿景

> **一句话定位**：Lumo 是一个本地优先、支持 WebDAV 的极致音乐播放器——极简、轻量、高性能、高颜值、数据完全属于你。

---

## 一、我们为什么要做 Lumo？

### 1.1 问题：音乐播放器的两极分化

音乐播放器市场呈现一种奇怪的割裂：

- **流媒体阵营**（Spotify / Apple Music / 网易云）：体验好，但你不"拥有"任何一首歌。停止付费、版权变动、服务停运——你苦心经营的歌单、收藏、播放记录随时烟消云散。更重要的是，你听了什么、什么时候听、听了多少次——这些极其私密的行为数据，全部被平台收集和分析。
- **本地播放器阵营**（Foobar2000 / MusicBee / AIMP）：数据完全属于你，但 UI 停留在上个时代，学习成本高，不支持 WebDAV 远程音乐库，无法跨设备同步。

> **Lumo 要填补的空白**：一个**既拥有流媒体级别的现代体验，又坚守本地优先哲学**的音乐播放器。

### 1.2 为什么是 WebDAV？

WebDAV 是一个开放标准的 HTTP 扩展协议，几乎所有 NAS（群晖、威联通）、网盘（坚果云、Nextcloud、Alist）都原生支持。选择 WebDAV 意味着：

- **无需自建服务端**：用户现有的 NAS / 网盘就是后端
- **数据主权不变**：文件依然存储在用户控制的设备上
- **多设备同步**：在家用 PC 听歌，出门用笔记本，数据库通过 WebDAV 自动同步
- **零额外成本**：不需要为 Lumo 部署任何东西

## 二、核心哲学：四项不可动摇的原则

### ① 本地优先（Local-First）

所有用户数据——曲库元数据、播放记录、歌单、收藏、偏好设置——全部存储于本地 SQLite 数据库中。断网状态下所有功能 100% 可用。

### ② 数据可迁移（Portable）

用户的全部数据 = 一个本地文件夹。拷贝到另一台电脑，Lumo 立刻恢复全部状态。

```
~/Lumo/
├── lumo.db              ← SQLite 主数据库（元数据、索引）
├── cache/               ← 封面缓存、音频缓存
└── config.json          ← 用户偏好配置
```

### ③ 零遥测（Zero Telemetry）

Lumo 不发送任何网络请求——没有匿名统计、没有崩溃上报、没有更新检查。唯一的网络活动是用户主动配置的 WebDAV 连接。

### ④ 非侵入（Non-Invasive）

Lumo 只建立索引指向用户的音乐文件，永远不移动、不复制、不修改用户的任何一个文件。卸载 Lumo 后，音乐文件完好无损。

## 三、产品功能蓝图

### 3.1 曲库管理

- 本地音乐目录扫描与增量索引
- 多格式音频标签解析（MP3 / FLAC / APE / WAV / OGG / AAC / WMA 等）
- 封面艺术自动提取与缓存
- 歌曲与文件分离建模（同一首歌可关联多个文件版本：FLAC 母带 + MP3 通勤版）
- 多维度浏览：艺术家 / 专辑 / 文件夹 / 流派 / 年份
- 重复文件检测与多音源归并
- 全局搜索（曲名、艺术家、专辑的模糊匹配与拼音搜索）

### 3.2 播放引擎

- 完整播放控制：播放 / 暂停 / 上一首 / 下一首
- 播放模式：顺序 / 随机 / 单曲循环 / 列表循环
- 无缝切歌（Gapless Playback）
- 播放队列管理（即时队列 + 稍后播放）
- 播放历史与统计
- 音量控制与音频输出设备选择
- 系统媒体键与全局快捷键支持

### 3.3 歌词

- LRC 歌词滚动同步显示
- 内嵌歌词提取（标签中的歌词字段）
- 外部 LRC 文件自动关联（同名文件匹配）

### 3.4 歌单

- 手动创建 / 编辑 / 删除歌单
- 智能歌单（基于条件自动聚合：最近添加、最常播放、按流派等）
- 收藏（快速标记喜欢的歌曲、专辑、艺术家）

### 3.5 WebDAV 远程音乐库

- 连接用户自有的 WebDAV 服务（NAS / 坚果云 / Nextcloud / Alist 等）
- 远程音乐目录浏览与元数据拉取
- 智能缓存策略：按需下载、LRU 淘汰
- HTTP Range 请求：大文件分片加载，无需完整下载即可播放
- 网络降级控制：弱网/断网时无缝切换到本地缓存
- 本地 + 远程统一曲库视图：在同一界面中无差别浏览和播放

### 3.6 跨设备数据同步（基于 WebDAV）

- 将 `lumo.db`（元数据库）和 `config.json`（配置）同步到 WebDAV
- 多设备间自动同步：歌单、收藏、播放进度、播放历史
- 冲突解决策略：基于时间戳的 Last-Write-Wins
- 仅同步元数据（KB 级别），不同步音频文件本身

### 3.7 UI 与体验

- **极简设计**：受 Apple Music / Spotify 启发的现代界面
- **沉浸式播放**：全屏 Now Playing 界面，封面艺术 + 歌词联动
- **双主题**：亮色 / 暗色主题，跟随系统或手动切换
- **流畅动画**：页面过渡、列表滚动、封面切换的微动画
- **响应式布局**：适配不同窗口尺寸
- **键盘友好**：空格播放、方向键切歌、Ctrl+F 搜索等

## 四、技术架构

### 4.1 技术栈

| 层 | 技术选型 | 选型理由 |
|---|---|---|
| **桌面容器** | Tauri v2 | 轻量（安装包 < 10MB）、安全、原生性能、跨平台（Windows / macOS / Linux）、未来支持移动端 |
| **前端框架** | Vue 3 (TypeScript) + Composition API | 现代、高效、生态成熟 |
| **样式引擎** | Tailwind CSS v4 | 原子化 CSS，主题系统灵活 |
| **状态管理** | Pinia | Vue 3 官方推荐，简洁高效 |
| **构建工具** | Vite | 极速 HMR，开发体验一流 |
| **后端语言** | Rust | 内存安全、极致性能、编译为原生二进制 |
| **数据存储** | SQLite (rusqlite, bundled) | 嵌入式、单文件、零配置 |
| **音频解码** | symphonia | 纯 Rust 实现，零外部依赖，支持 MP3/FLAC/WAV/OGG/AAC 等 |
| **音频播放** | rodio | 基于 symphonia 的高层播放接口 |
| **音频标签** | lofty | 支持 APE/FLAC/MP3/OGG 等多格式标签解析 |
| **WebDAV** | reqwest + quick-xml | 标准 HTTP 协议，灵活可控 |

## 六、设计原则

### 做什么

- ✅ 做**最好的本地音乐播放器**
- ✅ 做**最好的 WebDAV 音乐客户端**
- ✅ 做到**极致简单**——新用户 30 秒上手
- ✅ 做到**极致轻量**——安装包 < 10MB，内存占用 < 80MB
- ✅ 做到**极致高性能**——万首曲库秒级扫描，即点即播
- ✅ 做到**极致高颜值**——比肩 Spotify 的视觉体验

### 不做什么

- ❌ 不做视频播放
- ❌ 不做电子书阅读
- ❌ 不做照片管理
- ❌ 不做笔记编辑
- ❌ 不做密码管理
- ❌ 不做通用文件管理
- ❌ 不做 AI 功能（保持极致轻量）
- ❌ 不做社交功能
- ❌ 不做在线音乐搜索/下载

> **一个产品不是因为加了多少功能而伟大，而是因为砍掉了多少功能而伟大。**
