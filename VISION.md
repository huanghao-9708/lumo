# Lumo（轻音）— 产品愿景与战略蓝图

> **一句话定位**：Lumo 是一个以本地隐私数据自主权为核心哲学的个人数字资产管理平台，让用户在不出卖任何隐私的前提下，享受 AI 时代的智能化体验。

---

## 一、我们为什么要做 Lumo？

### 1.1 时代背景：数据主权意识的觉醒

我们正处在一个"数字资产"大规模流失的时代：

- **订阅疲劳**：Spotify、Apple Music、Netflix、Kindle Unlimited……用户每月支付大量订阅费，但"拥有"的东西为零——一旦停止付费，所有内容烟消云散。
- **服务停摆**：Google 关停了 Google Play Music、Stadia 等无数产品；国内网盘频繁关停或限速；任何云服务都可能在某一天发出"即将停止运营"的公告。
- **隐私裸奔**：你听了什么歌、看了什么视频、读了什么书——这些极其私密的行为数据，正在被各大平台收集、分析、交易，用于构建你的"数字画像"。
- **审查与下架**：云端的歌、书、电影随时可能因版权变动被下架，你曾经购买过的数字内容说没就没。

与此同时，一股**"数据回归本地"**的思潮正在全球技术社区涌动。Obsidian（本地笔记）的成功已经证明：**用户愿意为"数据完全属于自己"这件事买单。**

### 1.2 市场空白：一个没有被填补的生态位

我们调研了市面上所有现有方案，发现了一个清晰的产品定位空白：

| 维度 | Jellyfin / Plex | MusicBee / Foobar2000 | Calibre | **Lumo（我们）** |
|---|---|---|---|---|
| 零服务端，双击即用 | ❌ 需部署 Docker/NAS | ✅ | ✅ | ✅ |
| 音乐管理 | ⚠️ 基础 | ✅ 极强 | ❌ | ✅ |
| 视频管理 | ✅ 强项 | ❌ | ❌ | ✅ |
| 图书管理 | ⚠️ 插件勉强 | ❌ | ✅ 行业标准 | ✅ |
| WebDAV 原生支持 | ❌ | ❌ | ❌ | ✅ |
| 现代 UI | ✅ 尚可 | ❌ 老旧 | ❌ 极丑 | ✅ |
| 本地 AI 能力 | ❌ | ❌ | ❌ | ✅ |
| 隐私保护 | ⚠️ 自建才行 | ✅ | ✅ | ✅ |

> **结论**：市面上不存在一个"零服务端、本地优先、原生支持 WebDAV、同时覆盖音乐 + 视频 + 图书、具备本地 AI 能力、UI 精美"的轻量级桌面客户端。

## 二、核心哲学：五项不可动摇的原则

这五条原则是 Lumo 所有产品决策和技术决策的最高准则。任何新功能、新设计如果与这些原则冲突，必须让步。

### ① 本地优先（Local-First）

所有用户数据——元数据、播放记录、阅读进度、歌单、收藏、AI 生成的摘要和标签——全部存储于本地 SQLite 数据库中。没有任何云端依赖，断网状态下所有功能 100% 可用。

### ② 数据可迁移（Portable）

用户的全部数据 = 一个本地文件夹。拷贝这个文件夹到另一台电脑，Lumo 立刻恢复全部状态。无需"导出"、"备份"、"同步账号"等额外操作。

```
~/Lumo/
├── lumo.db              ← SQLite 主数据库（元数据、索引、向量）
├── cache/               ← 封面、缩略图、AI 生成的缓存
├── models/              ← 本地 AI 模型文件（可选下载）
└── config.json          ← 用户偏好配置
```

### ③ 零遥测（Zero Telemetry）

Lumo 不发送任何网络请求——没有匿名统计、没有崩溃上报、没有更新检查、没有"改善用户体验"的数据采集。唯一的网络活动是用户主动配置的 WebDAV 连接。

### ④ 原地索引，不动用户文件（Non-Invasive）

Lumo 只建立索引指向用户的媒体文件，永远不移动、不复制、不修改用户的任何一个字节。用户卸载 Lumo 后，他们的文件完好无损，就像 Lumo 从未存在过一样。

### ⑤ AI 在本地（On-Device AI）

所有 AI 推理均在用户设备上完成。用户的媒体内容、阅读习惯、搜索记录永远不会被发送到任何外部 AI 服务。

## 三、产品功能蓝图

### 3.1 媒体管理核心

#### 🎵 音乐模块

- 本地音乐目录扫描与增量索引
- 多格式音频标签解析（MP3 / FLAC / APE / WAV / OGG 等）
- 封面艺术自动提取与缓存
- 歌曲与文件分离建模（同一首歌可关联多个文件版本）
- 完整播放引擎（顺序 / 随机 / 单曲循环 / 歌单）
- LRC 歌词滚动显示
- 智能歌单与播放历史

#### 🎬 视频模块

- 本地视频目录扫描与分类索引
- 视频封面/缩略图智能截取
- 分层播放架构：
  - 第一层：HTML5 `<video>` 原生播放（MP4 / WebM）
  - 第二层：ffmpeg 快速转封装（MKV 等容器转 MP4，`-c copy` 不重编码）
  - 第三层：mpv/libmpv 原生引擎嵌入（远期，支持全格式）
  - 第四层：系统默认播放器兜底
- 播放进度记忆与续播
- 视频元数据管理

#### 📚 图书模块

- 本地电子书扫描与索引（EPUB / PDF / MOBI）
- 元数据自动提取（书名、作者、封面、出版信息）
- 内置阅读器：
  - EPUB：基于 epub.js 的沉浸式阅读体验
  - PDF：基于 pdf.js 的渲染
- 阅读进度跟踪与书签
- 书架式分类管理

#### 🖼️ 照片与相册模块

- 本地照片目录扫描与索引（JPEG / PNG / RAW / HEIF 等）
- EXIF 元数据自动解析（拍摄时间、GPS 地理位置、相机型号、光圈快门等）
- 时间线浏览：按年/月/日自动组织照片
- 地图浏览：基于 GPS 信息在离线地图上展示拍摄地点
- 相册管理：手动创建相册、智能相册（按日期/地点/人物自动聚合）
- 照片编辑：基础裁剪、旋转、滤镜（本地处理，不上传）
- AI 增强（远期）：人脸识别聚类、场景分类、以图搜图

#### 📝 笔记与文档模块

- 本地 Markdown 文件管理（类似 Obsidian 的文件夹 + 笔记模式）
- 内置 Markdown 编辑器（所见即所得 / 源码双模式）
- 双向链接与知识图谱（笔记之间的 `[[]]` 引用关系可视化）
- 全文搜索与标签系统
- 附件管理（图片、PDF 等附件就近存储在笔记目录中）
- 与图书模块联动：阅读笔记可直接关联到书籍
- AI 增强（远期）：笔记摘要、写作辅助、知识问答

#### 🔑 密码与敏感信息模块

- 本地加密数据库（AES-256-GCM 加密，主密码 + 可选密钥文件）
- 密码条目管理：网站账号、银行卡、软件许可证、私密笔记等
- 密码生成器：可配置长度、字符集、可读性
- 自动填充集成（远期，通过浏览器扩展）
- 数据库文件可通过 WebDAV 在多设备间同步（加密文件传输，服务端无法解密）
- 导入/导出：支持从 KeePass (.kdbx)、Bitwarden (JSON)、1Password (CSV) 等迁入
- **安全设计**：
  - 主密码不以任何形式存储，仅在内存中短暂存在
  - 应用锁屏/最小化后自动清除内存中的解密数据
  - Rust 实现核心加解密逻辑，避免 JavaScript 侧的内存安全风险

#### 📦 通用文件归档模块

- 任意类型文件的标签化管理（不限格式）
- 多维度组织：标签、分类、星标、颜色标记、自定义属性
- 快速预览：图片/PDF/文本/代码等常见格式的内置预览
- 全局搜索：文件名 + 标签 + 自定义属性的组合检索
- 智能收纳箱：一键导入待整理文件，稍后分类归档
- 文件去重检测（基于内容哈希）
- AI 增强（远期）：自动标签建议、文件内容摘要、相似文件发现

### 3.2 WebDAV 远程存储层

- 连接用户自有的 WebDAV 服务（NAS / 坚果云 / Nextcloud / Alist 等）
- 远程目录浏览与元数据拉取
- 智能缓存策略：按需下载、LRU 淘汰
- 网络降级控制：弱网/断网时无缝切换到本地缓存
- 多源统一视图：本地文件与远程文件在同一界面中无差别展示

### 3.3 本地 AI 能力层

> 所有 AI 功能均为**可选模块**，用户可以选择下载所需的模型，也可以完全不使用 AI 功能。

#### 语义搜索引擎

- 基于 Embedding 模型（如 all-MiniLM，约 80MB）的全库语义检索
- 支持自然语言查询："找那首去年夏天常听的慵懒爵士"、"关于日本旅行的所有内容"、"去年在京都拍的照片"
- 跨全部资产类型统一搜索（音乐 + 视频 + 图书 + 照片 + 笔记 + 文件）

#### 音频智能

- **Whisper 语音转写**（whisper-rs，75MB - 150MB）
  - 为没有歌词的歌曲自动生成歌词
  - 为视频自动生成字幕（SRT）
  - 视频内容语音索引（"找到视频里提到'预算方案'的位置"）
- **音频分类**：自动识别音乐的流派、情绪、BPM

#### 视觉智能

- **CLIP 模型**（约 350MB）：视频关键帧/封面智能选取、图像内容理解与标签、照片场景分类与以图搜图

#### 文本智能

- **小型语言模型**（Phi-4 Mini / Gemma 2B，量化后 1.5GB - 2GB）
  - 图书摘要自动生成
  - RAG 问答：基于书籍/笔记内容的本地问答（"这本书关于投资组合理论说了什么？"、"我的笔记里关于项目架构写了什么？"）
  - 智能推荐理由生成
  - 笔记写作辅助与自动摘要
  - 通用文件内容理解与标签建议

#### 个人推荐引擎

- 基于播放/阅读历史与内容 Embedding 的本地推荐算法
- 完全在本地计算，无"大数据画像"，只为用户一个人服务

## 四、技术架构

### 4.1 技术栈

| 层 | 技术选型 | 选型理由 |
|---|---|---|
| **桌面容器** | Tauri v2 | 轻量、安全、安装包极小、原生性能、支持移动端扩展 |
| **前端框架** | Vue 3 (TypeScript) + Composition API | 现代、高效、生态成熟 |
| **样式引擎** | Tailwind CSS v4 | 原子化 CSS，主题系统灵活 |
| **状态管理** | Pinia | Vue 3 官方推荐 |
| **构建工具** | Vite | 极速 HMR，开发体验一流 |
| **后端语言** | Rust | 内存安全、极致性能、编译为原生二进制难以逆向 |
| **数据存储** | SQLite (rusqlite, bundled) | 嵌入式、单文件、零配置、全球最广泛部署的数据库 |
| **音频解码** | symphonia + rodio | 纯 Rust 实现，零外部依赖 |
| **音频标签** | lofty | 支持 APE / FLAC / MP3 / OGG 等多格式标签解析 |
| **视频处理** | ffmpeg (Sidecar) + libmpv (远期) | 格式转封装 / 全格式原生播放 |
| **AI 推理** | candle / whisper-rs / ort / fastembed-rs | 纯 Rust 或 Rust 绑定，无需 Python 运行时 |
| **WebDAV** | reqwest + 自建 WebDAV 协议层 | 标准 HTTP 协议，灵活可控 |

### 4.3 安全设计

| 安全维度 | 措施 |
|---|---|
| **数据隔离** | 所有用户数据在 `~/Lumo/` 目录内，应用仅有对该目录和用户指定资产目录的访问权限 |
| **权限最小化** | Tauri v2 细粒度 Capabilities 机制，前端只能调用已授权的后端 API |
| **代码保护** | Rust 编译为原生二进制，核心逻辑（加解密、数据库操作、AI 推理）极难被逆向 |
| **网络隔离** | 默认零网络请求；WebDAV 连接由用户显式配置和授权 |
| **AI 隐私** | 所有模型本地运行，推理数据不出设备 |
| **密码安全** | AES-256-GCM 加密，主密码不存储，锁屏后自动清除内存中的解密数据 |
