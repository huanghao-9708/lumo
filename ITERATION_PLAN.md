# Lumo 迭代规划与现状总结 (基于真实的深度代码审查)

> **前言**：在进行了更深入的代码剖析后，我必须承认我之前**严重低估了 Lumo 当前的工程进度**。
> Lumo 的完成度高得惊人，这甚至不能算是一个雏形，而是一个**几近完成的 V1 版本**。无论是复杂的前后端交互，还是 WebDAV 的分片流播，都已经全部在代码中跑通了！

以下是基于代码现状（`src-tauri/src/lib.rs`、`player.ts` 等核心文件）和 `VISION.md` 整理的最真实的进度单与后续规划。

---

## 🟢 当前已完全实现的特性 (What's Already Done)

这部分功能不仅在后端写好了接口，而且已经**完整地绑定到了前端 Vue 的 UI 上**，形成了闭环：

### 1. 核心播放体验
- [x] **本地与 WebDAV 双引擎播放**：基于 `symphonia+rodio`，不仅能播本地文件，还能通过 `HttpRangeReader` 直接拉取 WebDAV 流播放。
- [x] **无缝切歌 (Gapless Playback)**：针对本地文件，底层已经实现了 `enqueue_next_file`。
- [x] **沉浸式控制台**：前端的 BottomPlayer 已经实现了进度条拖拽、精美的旋转音量物理旋钮、播放模式切换。
- [x] **系统级图片缓存分发**：Tauri 底层通过注册 `lumo://artwork` 自定义协议，配合信号量限流（Semaphore）和后台缩略图生成线程，极其优雅地解决了前端海量封面加载的卡顿问题。
- [x] **歌词系统**：已支持提取内嵌歌词并在 `LyricsView.vue` 中渲染。

### 2. 曲库与用户数据
- [x] **完整的数据库模型**：艺术家、专辑、曲目、源的管理，并自带分页查询。
- [x] **用户资产管理**：红心收藏（歌曲/专辑/艺术家）、手动创建歌单、添加整个文件夹到歌单、播放历史记录与统计，全套闭环均已上线。
- [x] **多源统一扫描**：支持本地增量扫描，支持 WebDAV `PROPFIND` 扫描与 HTTP Range 元数据提取，失效文件自动标记。

## 🚀 未来迭代计划 (True Future Iterations)

既然连 WebDAV 流播和播放器 UI 都已经闭环，接下来的迭代我们将**直奔高级体验**和**多端云同步**（即实现 `VISION.md` 中的 Phase 3 后半段、Phase 4 和 Phase 5）。

### 迭代一：WebDAV 体验极致化与网络降级
**目标**：把当前的 WebDAV 功能从“能用”变成“好用”，解决网络不稳定时的痛点。

1. **WebDAV 流播无缝切歌**：补全目前 `playback.rs` 中标注为 TODO 的 WebDAV 流式 Enqueue（Gapless Playback）功能。
2. **云端文件透明缓存**：用户在播放 WebDAV 的歌曲时，后台将其分片数据或完整文件异步存入本地缓存。同一首歌听第二遍时，实现“零网络请求”的本地秒开。
3. **断网/弱网降级控制**：
   - 探测系统网络状态。
   - 当设备离线时，UI 上自动将那些**纯云端且未被缓存**的曲目置灰，防止用户点击后一直转圈。

### 迭代二：跨设备数据同步 (真正的数据私有化多端漫游)
**目标**：实现 `VISION.md` 愿景中最核心的承诺——基于 WebDAV 的数据库同步，让电脑、笔记本甚至未来的手机端共享同一个数字大脑。

1. **数据上云**：将本地的 `lumo.db`（仅元数据，通常只有几 MB）和 `config.json` 手动或定期备份到用户配置的 WebDAV 根目录下的隐藏文件夹中（如 `.lumo_sync/`）。
2. **多端拉取与合并**：
   - 设备启动时，比对本地与云端数据库的时间戳或 Hash。
   - 如果云端较新，拉取云端数据库并实现热重载，瞬间同步在另一台设备上新建的歌单、红心收藏和播放历史。

### 迭代三：桌面端原生体验增强 (Phase 5 体验极致化)
**目标**：进一步逼近甚至超越 Spotify / Apple Music 的客户端体验。

1. **全局快捷键与媒体键支持**：接入操作系统的 Media Keys（多媒体键盘上的播放/切歌键），让 Lumo 在后台或最小化时依然能响应控制。
2. **多音源去重归并**：在 UI 层面，如果本地扫描到一个 `FLAC`，WebDAV 也扫到了同一首歌的 `MP3`，将它们归并为“一首歌的两个版本”，允许用户在播放时手动或根据网络环境自动切换音质。
3. **音频均衡器 (EQ)**：提供图形化 EQ 调节面板，满足发烧友需求。
4. **智能歌单 (Smart Playlists)**：基于 SQL 视图动态生成歌单，如“最近添加的 50 首歌”、“播放次数最多的流行乐”、“未听过的专辑”。

### 迭代四：跨平台与生态延伸
1. **macOS / Linux 适配**：解决文件路径斜杠、系统字体、窗口圆角等平台差异化细节。
2. **为移动端铺路**：梳理 Tauri v2 的移动端（iOS / Android）编译配置，确保目前的 Rust 库在移动架构下能够正常编译运行。
